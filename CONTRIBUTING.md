# Contributing

Thank you for helping improve Mend.

1. Open an issue before substantial product, safety, or architecture work.
2. Create a focused branch from `main`.
3. Preserve privacy, attribution, inclusivity, and crisis-routing boundaries.
4. Add or update tests for behavior changes.
5. Run `npm run verify` before opening a pull request.
6. Include the problem, approach, validation, risk, and screenshots for UI changes.

Clinical or crisis-adjacent claims require primary-source review and must not be
invented or silently generalized. Never commit credentials or private user
records. Security reports belong in the private process described in
`SECURITY.md`.
